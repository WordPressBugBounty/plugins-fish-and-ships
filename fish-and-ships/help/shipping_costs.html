Basic / Per ranges calculation

The shipping costs will be applied for each rule that matches the selection criteria.

(once):

The cost will be applied once, no matter how many products match the rule.

* [qty]:

The calculated cost will be the result of multiplying the amount of products that matches the selection criteria by the value entered.

* weight:

The calculated cost will be the result of multiplying the weight of products that matches the selection criteria by the value entered.

Be aware: if you need use rounded weight for calculatio, you must use ranges (see below).

* [group]:

The calculated cost will be the result of multiplying the number of matching groups (see the help section group-by by the value entered.

% (percentage):

The percentage will be calculated based on the product price. If there is a sale price, it will be used instead.

Regardless of the "Prices entered with tax" WooCommerce setting (can be tax included or tax excluded), price products will be taken as entered in the edition page.

Composite:

We can set the five previous calculations at once: once, * qty, * weight, * group and %. Those that are set with value 0 will not influence the calculation.

Ranges PRO:

With this advanced feature, you can set prices based on weight, volume, volumetric, quantity or dimensions, setting easily ranges of price, flat part and much more. It comes with an interactive preview.

You can read more about in this post: Shipping Rates by Weight Chart: Weight Ranges Made Easy

Be aware with group-by setting:
Keep in mind that the group-by setting (global or in each rule) will act only for the selection conditions (left side rules), not for shipping cost calculation, except for the * [group], of course. (see the group-by help section).

Not to be confused with the group-by setting within the Range settings dialog box: This will have the expected effect on the price calculation by ranges.
